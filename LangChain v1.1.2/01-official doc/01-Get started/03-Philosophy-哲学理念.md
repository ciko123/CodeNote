# 哲学理念

LangChain 致力于成为最简单的 LLM 应用开发框架，同时保持灵活性和生产就绪性。

## 核心信念
- **LLMs 是强大的新技术**：具有卓越的能力和潜力
- **外部数据增强能力**：LLMs 与外部数据源结合时更加强大
- **代理化应用趋势**：未来应用程序将越来越具有代理特性
- **早期发展阶段**：这一转型仍处于起步阶段
- **生产化挑战**：原型容易，但构建可靠的代理仍然困难

## 核心重点

### 1. 模型标准化
统一不同提供商的 API、参数和消息格式，让开发者轻松切换最新模型，避免供应商锁定。

### 2. 流程编排
让模型超越文本生成，支持与数据和计算交互的复杂流程。LangChain 简化工具定义和非结构化数据处理。

## 发展历程

### 2022-10 - v0.0.1
- Python 包发布，包含 LLM 抽象和链式处理
- 名称来源："Language" + "Chains"

### 2022-12
- 添加基于 ReAct 论文的通用代理

### 2023-01
- 支持 ChatCompletions API（消息列表格式）
- 发布 JavaScript 版本

### 2023-02
- LangChain Inc. 成立，目标："让智能代理无处不在"

### 2023-03
- 支持 OpenAI 函数调用 API

### 2023-06
- 发布 LangSmith 平台，提供可观察性和评估

### 2024-01 - v0.1.0
- 第一个稳定版本，注重生产就绪性

### 2024-02
- 发布 LangGraph，提供低级编排层
- 支持流式传输、持久执行、短期记忆、人在循环中

### 2024-06
- 超过 700 个集成，分离到独立包

### 2024-10
- LangGraph 成为复杂 AI 应用的首选方式
- 大多数链和代理标记为已弃用

### 2025-04
- 支持多模态模型 API（文件、图像、视频）

### 2025-10-20 - v1.0.0
- **重大重构**：所有链和代理替换为基于 LangGraph 的代理抽象
- **向后兼容**：通过 langchain-classic 包支持旧版本
- **消息格式标准化**：统一复杂输出类型（推理块、引用等）
